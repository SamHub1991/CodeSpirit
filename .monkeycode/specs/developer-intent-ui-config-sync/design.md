# 开发阶段意图识别与配置同步设计

## 方案

- 采用“意图结构体 + 配置面板 + 代码同步器”的三段式流程。
- 意图识别层把自然语言或模板选择转成结构化 `IntentSpec`。
- 渲染层根据 `IntentSpec` 生成预览 UI，并复用统一主题与布局规范。
- 配置层提供表单式编辑器，所有变更先落到 `IntentSpec`。
- 同步层把 `IntentSpec` 反向写回代码文件，并触发预览刷新。

## 核心数据模型

- `IntentSpec`
  - `pageType`
  - `components`
  - `layout`
  - `theme`
  - `bindings`
  - `actions`
  - `validation`
- `SyncResult`
  - `changedFiles`
  - `appliedPatches`
  - `warnings`

## 关键流程

1. 开发者输入需求或打开现有配置。
2. 意图识别器抽取结构化字段并计算置信度。
3. 配置面板展示当前结构，允许补全和调整。
4. 预览渲染器按统一样式生成界面。
5. 保存时同步器生成代码补丁并写入目标文件。
6. 代码变更后，热刷新预览并回传变更结果。

## 代码同步策略

- 以配置为源数据，代码为可生成产物。
- 在文件中划分生成区和手工区，更新时只写生成区。
- 同步前先生成 diff，供开发者确认后应用。
- 同步结果写入本地元数据，支持重载恢复。

## 界面结构

- 左侧：意图与配置树。
- 中间：实时预览画布。
- 右侧：属性编辑与冲突提示。
- 底部：同步日志和变更文件列表。

## 验证点

- 配置修改后，预览与代码文件同时更新。
- 冲突配置会被明确标记。
- 响应式布局在窄屏下可正常浏览。
