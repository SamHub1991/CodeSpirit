# 开发阶段意图识别与配置同步任务清单

## 阶段 1：意图模型与配置入口

- [ ] 定义 `IntentSpec` 和 `SyncResult` 的结构，覆盖页面类型、组件、布局、主题、绑定、动作和校验。
- [ ] 在开发者入口补充意图输入或配置草案入口，支持模板选择和自然语言转结构化配置。
- [ ] 为低置信度识别补充提示区，保留已识别字段并请求开发者补全。

## 阶段 2：预览渲染与界面层

- [ ] 建立统一预览渲染管线，把 `IntentSpec` 映射为页面、组件和响应式布局。
- [ ] 落实漂亮界面的基础视觉规范，包括层级、间距、字体、状态提示和桌面/移动适配。
- [ ] 在页面中加入配置面板、预览画布、属性编辑区和同步日志区。

## 阶段 3：配置编辑与冲突处理

- [ ] 把配置面板和现有 MVVM state 连通，支持主题、布局、字段映射、数据源、交互和校验的即时修改。
- [ ] 实现配置冲突检测与修正建议展示。
- [ ] 在配置变更时触发局部预览刷新，缩短反馈回路。

## 阶段 4：代码同步与恢复

- [ ] 实现配置到代码的同步器，生成受影响文件的最小补丁。
- [ ] 约定生成区与手工区的边界，保证同步时保留人工编辑内容。
- [ ] 记录变更来源、时间和影响范围，并支持重新打开项目时恢复最近一次成功同步状态。

## 阶段 5：模板同步与验证

- [ ] 同步更新 `src/CodeSpirit.LibraryManagement` 与 `src/Templates/CodeSpiritVsixTemplate/ProjectTemplates/CodeSpirit.LibraryManagement` 的对应实现。
- [ ] 补充验证步骤，确认配置修改后预览与代码文件同时更新。
- [ ] 检查窄屏和桌面宽度下的布局表现，确认界面可用性。
