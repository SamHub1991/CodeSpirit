2. 自动扫描与注册机制
利用.NET 10的源生成器（Source Generator），在编译时自动扫描带有特定Attribute的程序集并生成注册代码：
// AutoServiceRegistrar.cs —— 自动扫描注册器
public static class AutoServiceRegistrar
{
    public static IServiceCollection AddCodeSpiritAutoServices(this IServiceCollection services, Assembly assembly)
    {
        // 扫描所有带有[Service]注解的类
        var serviceTypes = assembly.GetTypes()
            .Where(t => t.IsClass && !t.IsAbstract && t.GetCustomAttribute<ServiceAttribute>() != null);

        foreach (var type in serviceTypes)
        {
            var attr = type.GetCustomAttribute<ServiceAttribute>()!;
            var serviceType = attr.ServiceType ?? type.GetInterfaces().FirstOrDefault() ?? type;
            
            services.Add(new ServiceDescriptor(serviceType, type, attr.Lifetime));
        }

        return services;
    }
}
3. 自动配置模块
借鉴Spring Boot的@EnableAutoConfiguration机制，CodeSpirit通过条件化配置实现自动装配：
// AutoConfigurationAttribute.cs —— 自动配置注解
[AttributeUsage(AttributeTargets.Class)]
public class AutoConfigurationAttribute : Attribute
{
    public Type? ConditionalOnBean { get; set; }
    public string? ConditionalOnClass { get; set; }
}

// CodeSpiritAutoConfiguration.cs —— 框架入口
public static class CodeSpiritAutoConfiguration
{
    public static WebApplicationBuilder AddCodeSpirit(this WebApplicationBuilder builder)
    {
        var assembly = Assembly.GetCallingAssembly();
        
        // 1. 自动注册服务
        builder.Services.AddCodeSpiritAutoServices(assembly);
        
        // 2. 自动配置模块
        var configModules = assembly.GetTypes()
            .Where(t => t.GetCustomAttribute<AutoConfigurationAttribute>() != null);
        
        foreach (var module in configModules)
        {
            var instance = Activator.CreateInstance(module) as ICodeSpiritModule;
            instance?.Configure(builder.Services, builder.Configuration);
        }
        
        // 3. 自动配置中间件
        builder.Services.AddCodeSpiritMiddleware();
        
        return builder;
    }
}

五、AOP切面编程：注解式拦截
借鉴Spring AOP的@Transactional、@Cacheable等注解，CodeSpirit通过Castle.DynamicProxy实现声明式切面编程：
// 事务注解 —— 类似Spring的@Transactional
[AttributeUsage(AttributeTargets.Method)]
public class TransactionalAttribute : Attribute { }

// 缓存注解 —— 类似Spring的@Cacheable
[AttributeUsage(AttributeTargets.Method)]
public class CacheableAttribute : Attribute
{
    public string CacheKey { get; set; } = "";
    public int ExpirationSeconds { get; set; } = 300;
}

// AOP拦截器实现
public class TransactionInterceptor : IInterceptor
{
    private readonly AppDbContext _dbContext;
    
    public void Intercept(IInvocation invocation)
    {
        if (invocation.Method.GetCustomAttribute<TransactionalAttribute>() != null)
        {
            using var transaction = _dbContext.Database.BeginTransaction();
            try
            {
                invocation.Proceed();
                transaction.Commit();
            }
            catch
            {
                transaction.Rollback();
                throw;
            }
        }
        else
        {
            invocation.Proceed();
        }
    }
}

基于搜索结果，我为您设计一个融合三者优势的现代化框架方案。这个方案的核心目标是：保留.aspx文件的简洁性与熟悉感，同时引入MVVM模式、模块化热插拔和现代化开发体验。
一、方案概述
现代Web开发面临的核心矛盾是：传统Web Forms的易用性与现代化架构（MVVM、模块化、热更新）之间的矛盾。您提出的方案正是解决这一矛盾的创新尝试。
二、核心架构设计
1. 三层次架构
借鉴DotVVM的MVVM模式、CodeBehind的模块化热插拔机制和传统Web Forms的.aspx文件格式，设计如下架构：
表示层（.aspx文件） → ViewModel层（C#类） → 业务逻辑层（服务/控制器）
2. 文件结构设计
/Pages/
  Customer.aspx              # 视图文件（保留.aspx后缀）
  CustomerViewModel.cs       # ViewModel类（继承DotVVM的DotvvmViewModelBase）
  CustomerController.cs      # 控制器（可选，处理请求前后逻辑，借鉴CodeBehind）
3. 页面指令简化
借鉴CodeBehind的极简指令风格：
<%@ Page Controller="CustomerController" ViewModel="CustomerViewModel" %>
结合Web.config全局配置后，可进一步简化为：
<%@ Page ViewModel="CustomerViewModel" %>

3. 渐进式迁移能力
DotVVM支持与现有Web Forms应用共存运行，逐步将页面从Web Forms迁移到新框架：

“您可以安装DotVVM NuGet包到Web Forms应用中，它将与项目中的ASPX页面共存运行。从部署角度看，没有变化——仍然是同一个ASP.NET应用部署到服务器。”

四、技术实现要点
1. 自定义页面解析器
需要开发自定义的PageParser来处理简化后的指令语法，识别Controller和ViewModel属性。
2. 依赖注入集成
借鉴ASP.NET Core的依赖注入机制，支持在ViewModel和Controller中注入服务。
3. 路由管理
支持基于约定的路由自动推断（如/Customer自动映射到CustomerController）。
4. 生命周期管理
结合DotVVM的页面生命周期（Init → Load → PreRender）和CodeBehind的请求拦截机制（BeforeLoad/AfterLoad）。
五、开发体验优化
1. 默认命名空间配置
在Web.config中统一配置：
<system.web>
  <pages>
    <namespaces>
      <add namespace="MyApp.ViewModels" />
      <add namespace="MyApp.Controllers" />
    </namespaces>
  </pages>
</system.web>
2. 默认ViewModel推断
遵循“约定优于配置”原则：如果Controller名称与ViewModel名称存在约定关系（如CustomerController对应CustomerViewModel），可以自动推断。
3. Visual Studio扩展支持
参考DotVVM的VS扩展，提供语法高亮、IntelliSense和错误检查。
4. 新手上手路径
提供页面模板、ViewModel模板、常用表单模板和示例工程，让开发者从页面、状态和命令三部分快速理解框架。
提供统一验证、错误回显、提交反馈和列表分页等常用模式，减少首次搭建复杂页面的学习成本。
5. 复杂界面组合能力
页面支持区域化渲染、组件化布局、局部刷新和条件显示，适合仪表盘、管理后台和多步骤流程页面。
ViewModel 支持嵌套状态、命令驱动交互和验证状态聚合，便于组织大页面中的复杂行为。

第一阶段（1-2周）：搭建原型，实现基本的.aspx页面解析和MVVM绑定
第二阶段（2-4周）：实现模块化热插拔机制，支持动态加载Controller和ViewModel
第三阶段（2-4周）：完善开发工具链，包括VS扩展和代码片段
